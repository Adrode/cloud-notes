#JWT
- **JWT** - JSON Web Token
	- jest to zakodowany JSON z podpisem
	- składa się z:
		- header
			- "typ": "JWT", "alg": "HS256"
		- payload
			- dane dotyczące usera, ID, czas wygaśnięcia tokena, czas uzyskania tokena
		- signature
			- to jest podpis, tworzony przez funkcję haszującą, do której przekazany jest header i payload
	- **one nie są zaszyfrowane, tylko zakodowane przez Base64**; znaczy to tyle że każdy może podejrzeć token

W projekcie z FastAPI potrzebne są paczki (instalacja przez pip):
- python-jose\[cryptography]
- passlib\[bcrypt]
- python-multipart
Są to dependencies potrzebne do obsługi JWT


**bcrypt** - funkcja hashująca do przechowywania haseł
- bierze hasło, salt (losowe bajty), cost factor (liczba rund / koszt obliczeń), wykonuje algorytm i tworzy **hash**
- jest **jednokierunkowy** (czyli tylko zmienia plain hasło w hash, w drugą stronę się nie da) i **zawiera w sobie salt i cost**
`bcrypt_context.hash(plain_password)` - tworzy hash na bazie przekazanego hasła
`bcrypt_context.verify(plain_password, hashed_password)` - przepuszcza plain_password przez algorytm i sprawdza zgodność z hashed_password; bierze salt z hashed_password do przeliczenia
- jest to bardzo bezpieczne, bo odgadnięcie hasła polega na próbie odgadnięcia plain hasła, hashowaniu go i porównywaniu wyników z wykradzionym hashem, więc jest to bardzo czasochłonne

**timedelta** - obiekt z modułu `datetime`, reprezentuje ilość czasu którą można dodać lub odjąć od daty, np. `datetime.now() + timedelta(minutes=20)`

**Depends()** - robi **dependency injection**, czyli wstrzykuje zależności
- jak Depends() jest podane w argumencie endpointu, to wywoływana jest funkcja przekazana w Depends() i jej wynik jest wstrzykiwany do argumentu funkcji
	- wstrzyknięty może być też generator (jak `get_db`), asynchroniczne funkcje, itd.
- każdy endpoint w projekcie FastAPI + SQLAlchemy potrzebuje zależności od Session (`db: Session = Depends(get_db)`) , inaczej trzeba by było pisać to ręcznie

**`response_model`** - jest to "sitko", które zamienia to co zwraca endpoint / funkcja zgodnie z przekazaną do response_model schemą
- ```python
  @router.get("/me", response_model=UserSchema)
  def get_me(current_user: models.User = Depends(get_current_user)):
	  return current_user
  ```
	- tutaj funkcja przyjmuje wszystkie dane z modelu User, ale znajduje się tam `hashed_password`, którego nie chcę zwracać, więc żeby obsłużyć zwrócenie tylko konkretnych danych tworzy się schemę, jak `UserSchema` która przepuszcza tylko np. id, username, dzięki czemu endpoint zwróci tylko id i username

**`from_attributes`** - jeśli funkcja zwraca SQLAlchemy object, ale potrzebuję użyć `response_model` w tej funkcji, to jest problem bo Pydantic obsługuje dict'y, a na SQLAlchemy object wywoła błąd; ze względu na to używa się w schemie:
- ```python
  class UserSchema(BaseModel):
	  ...
	  
	  class Config:
		  from_attributes: True
  ```
	- czyli to pozwala na SQLAlchemy na zrobienie dict na bazie atrybutów obiektu SQLAlchemy; rozwiązuje to wspomniany błąd

## Kod JWT Authentication
#JWT
- `auth.py` - funkcje obsługujące hashowanie haseł, autentykację usera, tworzenie i weryfikowanie tokenów
	- wymagane rozszerzenia `python-jose` (dla jwt), `passlib` (dla bcrypt, czyli hashowania haseł)
```python
from datetime import datetime, timedelta
from jose import JWTError, jwt
from passlib.context import CryptContext
from fastapi import Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer
from sqlalchemy.orm import Session
from database import get_db
import models

# JWT config
SECRET_KEY = "dev-secret-key_to_change_in_production"
	# losowy string, powinien niejawny w strukturze plików
ALGORITHM = "HS256"
	# algorytm do podpisywania tokenu jwt
ACCESS_TOKEN_EXPIRE_MINUTES = 30

# hashowanie hasła
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# depenedency do wyciągania tokenu z headera Authorization
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="auth/login")
    # zapisuje token w formie stringa, jak go nie ma to błąd 401
    # tokenUrl służy do dokumentacji Swaggera (wskazuje endpoint pod którym uzyskuje się autoryzację); przez backend nie jest to wykorzystywane

# PASSWORD
# -------------------
def hash_password(password: str) -> str:
  return pwd_context.hash(password)
    # hashowanie przekazanego hasła
    # używane przy rejestrowaniu usera

def verify_password(plain: str, hashed: str) -> bool:
  return pwd_context.verify(plain, hashed)
    # weryfikacja plain_password z hashem
    # używane min. przy logowaniu usera i nadawaniu tokenu
# -------------------

# USERS
# -------------------
def get_user_by_email(db: Session, email: str):
  user = db.query(models.User).where(models.User.email == email).first()
  return user
	# funkcja pomocnicza, do sprawdzenia czy user istnieje

def authenticate_user(db: Session, email: str, password: str):
  user = get_user_by_email(db, email)
  if not user or not verify_password(password, user.hashed_password):
    return None
  return user
    # funkcja pomocnicza do zweryfikowania hasła usera lub jego istnienia
# -------------------

# JWT
# -------------------
def create_access_token(data: dict, expires_delta: timedelta | None = None) -> str:
  to_encode = data.copy() # data.copy() robi się, żeby nie modyfikować oryginalnych danych z data, czyli oryginalnego słownika przekazanego w argumencie funkcji
  expire = datetime.now() + (expires_delta or timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES))
  to_encode.update({"exp": expire})
  return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    # funkcja, która tworzy token; przyjmuje data, secret_key i algorytm
	    # data zawiera {sub: "email / username / id", exp: "date"}
		    # w sub musi być wartość która jednoznacznie określa usera (unique)
		    # w exp podaje się datę wygaśnięcia tokenu
		    # ALGORITH, tutaj HS256, bierze data i secret_key, na podstawie czego tworzy podpis
    # ta funkcja używana jest przy logowaniu, nadaje token
# -------------------

# CURRENT USER
# -------------------
def get_current_user(
  token: str = Depends(oauth2_scheme),
  db: Session = Depends(get_db)
):
  credentials_exception = HTTPException(
    status_code=401,
    detail="Could not validate credentials",
    headers={"WWW-Authenticate": "Bearer"}
  )
  try:
    payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
    email: str = payload.get("sub")
    if email is None:
      raise credentials_exception
  except JWTError:
    raise credentials_exception
	# jeżeli token jest niepoprawny, to po jwt.decode będzie błąd JWTError
	# również jeśli exp już minęło, też będzie błąd JWTError
  
  user = get_user_by_email(db, email)
  if user is None:
    raise credentials_exception
  if not user.is_active:
    raise HTTPException(
      status_code=401,
      detail="Account is inactive",
      headers={"WWW-Authenticate": "Bearer"}
    )
  return user
	# funkcja, która weryfikuje poprawność tokenu i przydziela dostęp
	# tę funkcję wstrzykuje się (Depends) do endpointów, w których wymagam uwierzytelnienia użytkownika
# -------------------
```

- `api/auth.py` - endpointy dotyczące autentykacji
```python
from fastapi import APIRouter, Depends, HTTPException
from fastapi.security import OAuth2PasswordRequestForm
from sqlalchemy.orm import Session
import models, schemas, auth
from database import get_db

router = APIRouter()

@router.post("/register", response_model=schemas.User)
def register(
  user_data: schemas.CreateUser,
  db: Session = Depends(get_db)
):
  if auth.get_user_by_email(db, user_data.email):
    raise HTTPException(
      status_code=400,
      detail="Email not available"
    )
  
  user = models.User(
    email=user_data.email,
    name=user_data.name,
    hashed_password=auth.hash_password(user_data.password)
  )
  db.add(user)
  db.commit()
  db.refresh(user)
  return user
	# endpoint, który tworzy usera z hashed password

@router.post("/login", response_model=schemas.Token)
def login(
  form_data: OAuth2PasswordRequestForm = Depends(),
  db: Session = Depends(get_db)
):
  user = auth.authenticate_user(db, form_data.username, form_data.password)
  if not user:
    raise HTTPException(
      status_code=401,
      detail="Incorrect email or password",
      headers={"WWW-Authenticate": "Bearer"}
    )
  token = auth.create_access_token(
    data={"sub": user.email}
  )
  return {"access_token": token, "token_type": "bearer"}
	# endpoint, który loguje usera; nadaje emaila do sub, przekazuje to do funkcji create_access_token, dzięki czemu user ma aktywną sesję z tokenem
	
@router.get("/me")
def get_me(current_user: models.User = Depends(auth.get_current_user)):
  return current_user
	# endpoint, który ma wstrzyknięte (Depends) get_current_user, czyli weryfikuje poprawność tokenu usera
```
