- Aby połączyć obie technologie podstawowo należy utworzyć (najlepiej w osobnych plikach), np:
	- `database.py` - create_engine i SessionLocal
	- ```python
	engine = create_engine(...)
	
	SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
	# autocommit=False - sam robię db.commit() tam gdzie potrzebuję
	# autoflash=False - autoflush czasem sam robi niespodziewane zapytania do bazy danych, przy False sam używam Flush gdy potrzebuję
	# bind=engine
	
	def get_db():
		db = SessionLocal()
		try:
			yield db
		finally:
			db.close()
		```
		- dzięki SessionLocal w FastAPI każdy request będzie miał własną sesję
			- to jest **dependency dla FastAPI**
	- `main.py` - FastAPI i endpointy
	- ```python
	from fastapi import FastAPI, Depends # Depeds związane z dependency z SessionLocal
	from sqlalchemy.orm import Session
	from models import Base, User, Note
	from database import enigne, get_db
	
	Base.metadata.create_all(bind=engine)
	
	app = FastAPI()
	
	@app.get("/users")
	def get_users(db: Session = Depends(get_db)):
		users = db.query(User).all()
		return users
	  ```
		- wykorzystuje się tutaj `Depends`, które w argumencie przyjmuje funkcję `get_db`, aby dla każdego requesta tworzyć nową sesję automatycznie
			- bez tego trzeba by było ręcznie otwierać i zamykać sesję
				- ```python
				  def get_users():
					  db = SessionLocal()
					  users = db.query(User).all()
					  db.close()
					  return users
				  ```
				- powyższe jest **niewłaściwym sposobem!**
	- `models.py` - modele tabel
		- czyli standardowe modele / klasy tabel, które potem importuję do main.py i dopiero tam robię `Base.metadata.create_all(bind=engine)`

## Jak obsłużyć błąd wywołany przez constraint?
#IntegrityError
- ```python
  from sqlalchemy.exc import IntegrityError
  from fastapi import HTTPException
  
  @app.post("/users")
  def add_user(email: str, name: str, db: Session = Depends(get_db)):
	  try:
		  new_user = User(email=email, name=name)
		  db.add(new_user)
		  db.commit()
		  db.refresh(new_user)
		  return new_user
	  except IntegrityError:
		  db.rollback()
		  raise HTTPException(status_code=404, detail="Email already exists")
```
	- powyższe obsługuje np. Unique constraint, gdy próbujemy dodać do tabeli coś, co kłóci się z Unique

## Jak działa APIRouter?
#APIRouter
- `APIRouter` to obiekt, który zbiera definicje endpointów, ale nie uruchamia serwera
	- czyli `FastAPI()` to aplikacja (serwer HTTP), a `APIRouter()` to pojemnik na trasy
	- czyli jak chcę trzymać endpointy w innym pliku niż `main.py`, to łączę pliki przez `APIRouter`; wtedy w `main.py`:
	- ```python
	  from fastapi import FastAPI
	  from models import Base
	  from database import engine
	  from api import users, notes # endpointy
	  
	  Base.metadata.create_all(bind=engine)
	  app = FastAPI()
	  
	  app.include_router(users.router, prefix="/users", tags=["Users"])
	  app.include_router(notes.router, prefix="/notes", tags=["Notes"])
	  # .include_router dołącza router do app
	  # prefix ustawia początek ścieżki do endpointu
	  # tags dodaje wspólne taki, co grupuje dokumentację 
	  ```
	- pozostałe pliki, np. `api/notes`:
	- ```python
	  from fastapi import APIRouter
	  ...
	  
	  router = RouterAPI()
	  
	  @router.get("/")
	  def get_notes(db: Session = Depends(get_db)):
		  ...
		  
	  @router.get("/{id}") # wtedy z prefixem ścieżka będzie "/notes/{id}"
	  def get_note(id: int, db: Session = Depends(get_db)):
		  ...
	  ```
	- można też dodawać opisy endpointów:
	- ```python
	  @router.post(
		  "/notes",
		  summary="Create a new note",
		  description="Creates a note assigned to a specific user.",
		  response_description="The created note object"
	  )

		  ...
	  ```
