#pip #virtual_environment 
## Czym jest pip?
#pip
- **pip** - system zarządzania pakietami
	- `pip` - działa jak help
	- `pip --version` - zainstalowana wersja
	- `pip install --upgrade pip` - update
	- `pip list` - lista zainstalowanych paczek
	- `pip list --outdated` - lista paczek, które można zaktualizować
	- `pip install "package"` - instalacja paczki

## Czym jest i jak działa virtual environment?
#virtual_environment #venv
- **Virtual environment** - lokalizacja tworząca środowisko; pozwala na zachowanie zależności, wersji oraz paczek, zapobiegając konfliktom pomiędzy projektami
	- virtual env. standardowo tworzy się wewnątrz katalogu projektu, którego ma dotyczyć
	- w terminalu, będąc w odpowiednim katalogu, należy użyć komendy
		- `python -m venv env`, gdzie `env` to nazwa środowiska, ale przyjęło się że nazywa się je "`env`"

## Jak korzystać z venv?
#venv 
- `source env/bin/activate` - aktywuje środowisko
	- `pip list` - lista paczek w środowisku
	- `pip install "package"` - instalowanie paczki w środowisku
	- `pip freeze > requirements.txt` - zapisuje nazwy zainstalowanych paczek do pliku requirements.txt
	- `pip install -r requirements.txt` - mając już utworzone nowe środowisko w tym samym katalogu, w którym znajduje się plik requirements, możemy skorzystać z wymienionej komendy żeby zainstalować paczki wymienione w pliku
- `deactivate` - deaktywuje (gdy jesteśmy w aktywnym środowisku)
