#constraints
## Jak działa UNIQUE?
#unique 
- **`UNIQUE`** - to constraint dla kolumny, który oznacza że każde value w tej kolumnie musi być unikalne; nie jest to to samo co Primary Key
- **`ALTER TABLE person ADD UNIQUE (email);`** - dodanie constrainta unique do istniejącej tabeli
- **`ALTER TABLE person ADD CONSTRAINT unique_email_address UNIQUE (email);`** - jak wyżej, tylko własna nazwa constrainta 

## Jak działa CHECK?
#check
- **`CHECK`** - constraint, który pozwala na określenie jakie wartości mogę być przyjęte w danej kolumnie
- **`ALTER TABLE person ADD CONSTRAINT gender_constraint CHECK (gender = 'Female' OR gender = 'Male');`** - dopuszczalne będą tylko przekazane wartości, jakakolwiek inna wartość wyrzuci błąd
